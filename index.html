01URL-based content

Bring the link.
Omnius does the rest.

Nothing is hosted or bundled by Omnius. You add what you want to watch by URL — and the server fetches metadata, organizes it, and streams it on demand.

01

03Run the server

Self-host in
one command.

Omnius ships as a public container image and a single static binary, straight from the release feed. Your data, your machine.

zshDocker 
$ docker run -d --name omnius \
    -p 8080:8080 \
    -v omnius-data:/app/data \
    -e ADMIN_PASSWORD=changeme \
    ghcr.io/omniusrepos/omnius-server:latest

# → admin at http://localhost:8080/admin
zshBinary · any Linux 
$ curl -L …/releases/latest/download/omnius-linux-amd64 \
    -o omnius
$ chmod +x omnius
$ ADMIN_PASSWORD=changeme ./omnius

# → listening on :8080 · arm64 build too
Then: open /admin, add a movie by IMDB code or a live-TV M3U URL, and point the app at your server. Data lives in /app/data — back it up.
